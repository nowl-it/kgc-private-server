# Deploy & Run

How to bring the server up, connect a device, and push a change to the client.

## Servers — two uvicorn processes

The client speaks HTTPS; the SSL-bypass patch in the client accepts any cert, so a self-signed
cert is fine. Run **both** listeners (HTTP for convenience/CDN, HTTPS for the real client):

```bash
cd server
uvicorn server:app --host 0.0.0.0 --port 8080                                   # HTTP
uvicorn server:app --host 0.0.0.0 --port 8443 --ssl-keyfile key.pem --ssl-certfile cert.pem  # TLS
```

- `cert.pem` / `key.pem` are generated by `setup.py` (gitignored — never ship the private key).
- Both processes cache `real_cdn/*` file bytes **at import** (`_CDN_FILES` in `server.py`). After a
  master-data/bundle change you **must restart both** to serve the new bytes.
- Player state (`state/*.json`) is **not** cached — `load_state()` reads it per request. State edits
  are live without a restart.
- Dashboard (optional): `python3 dashboard.py` → http://localhost:8081/ (Battle Tracker + Admin).

### Restarting cleanly

Background jobs die with the shell, so launch each detached and on its own:

```bash
pkill -f "uvicorn server:app"
cd server && setsid uvicorn server:app --host 0.0.0.0 --port 8080 \
  > /tmp/kgc_server.log 2>&1 < /dev/null & disown
cd server && setsid python3 -m uvicorn server:app --host 0.0.0.0 --port 8443 \
  --ssl-keyfile key.pem --ssl-certfile cert.pem > /tmp/kgc_tls8443.log 2>&1 < /dev/null & disown
ss -ltn | grep -E ':8080|:8443'      # confirm both listening
```

Logs: `/tmp/kgc_server.log` (:8080), `/tmp/kgc_tls8443.log` (:8443). The TLS log is where you watch
the real client's requests (`POST /auth/login`, `GET /player`, `[GAME/START] body=…`, `CDN GET /patch/…`).

## Connect a device (uniform path — works on every target)

Bake `127.0.0.1` into the client and route the device's `:443` back to your `:8443` with `adb reverse`.
No root, no hosts-file edit, no privileged port. Identical on redroid / BlueStacks / LDPlayer / a real
USB phone — all support `adb reverse`.

| Target | adb connect | Typical serial |
|--------|-------------|----------------|
| redroid (Linux/Docker) | `adb connect localhost:5556` | `localhost:5556` |
| BlueStacks | enable ADB, `adb connect 127.0.0.1:5555` | `127.0.0.1:5555` |
| LDPlayer | `adb connect 127.0.0.1:5555` | `127.0.0.1:5555` |
| Real phone | USB debugging, plug in | from `adb devices` |

```bash
# 1. Build + install the side-by-side test app (com.nowl.castle, "King Bug Castle")
ADB_SERIAL=<serial> python3 server/rebuild_arm64_mod.py --host 127.0.0.1

# 2. Route :443 -> :8443 (re-run after replug / emulator restart — it is per-connection)
adb -s <serial> reverse tcp:443 tcp:8443
adb -s <serial> reverse tcp:80  tcp:8080     # optional (CDN is https, usually not needed)

# 3. Launch
adb -s <serial> shell am start -n com.nowl.castle/com.awesomepiece.castle.MainActivity
```

Healthy boot in logcat: `LibMainWrap → libxigncode.so → libmain_real.so → Unity il2cpp`, then
`Hooked BattleManager.Update` / `Hooked PostListItem.Set`. Then the TLS log fills with `200 OK`.

For **remote players** (no adb) you bake a public host instead — see [../SHARE.md](../SHARE.md).

## Push a master-data change to the client

Any edit under `scratchpad/xml_live/*.xml` (stage spawns, skins, text, `MinVersion`) reaches the client
only through the CDN xml bundle:

```bash
python3 server/rebuild_xml_bundle.py         # rebuilds real_cdn/xml + updates AssetHash.txt
# restart BOTH uvicorns (see above) so they re-cache real_cdn/
adb -s <serial> shell "rm -rf /sdcard/Android/data/com.nowl.castle/files/UnityCache"   # optional
adb -s <serial> shell am force-stop com.nowl.castle
```

The `AssetHash.txt` for `xml` changes on every rebuild → the client's CDN check detects the diff and
re-downloads the bundle on next launch. Details: [cdn-master-data.md](cdn-master-data.md).

## Server-only state change

Editing `state/player.json` (or `state/players/<pid>.json`) takes effect on the client's **next fetch**
of the relevant endpoint — no restart, no re-download. See [save-editing.md](save-editing.md).
